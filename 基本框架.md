## 一、界面布局

### 1. 左侧侧栏（宽度 300px）

- **📅 追番时间轴**（`WatchCalendar.tsx`）  
  基于 Excel 中「首刷时间」列，按年-月分组，纵向时间轴展示。支持最新/最早排序切换，点击条目弹出详情。

- **📊 维度排序**（`Sidebar.tsx`）  
  8 个维度（总评不参与排序）+ BGM评分 + 番名，以标签形式切换。选中后展示全部番剧排名列表（可滚动），前 3 名粉色高亮，支持正序/倒序切换。点击维度标签联动主网格排序（番名排序保持在当前分类，其余维度自动切换到「看过」）。默认选中「总评」。

- **🏷️ Tag 管理**（`Sidebar.tsx`）  
  自动收集全部番剧标签，按使用次数降序 + 中文拼音排序展示。功能：
  - **筛选**：点击标签 → 主网格仅显示包含该标签的番剧（再次点击取消）
  - **编辑模式**：标题栏右侧 ✏️ 按钮进入编辑模式后，可重命名（全局生效）、删除（全局移除）、新增预设标签（计数 0，可在详情面板中使用）
  - **批量添加**：点击「批量添加」→ 侧栏标签多选 + 主网格卡片多选 → 一键将选中标签添加到选中番剧
  - 所有编辑操作（增删改）同步写回 Excel

- **⚙️ 设置**  
  - 📤 导入 Excel：选择 .xlsx 文件替换内存数据
  - 📥 导出 Excel：下载 `番评分_导出.xlsx`
  - 📐 维度管理：增删改维度 + 权重滑块
  - 🖼️ 图片高度：滑块 200-800px，默认 360

### 2. 主界面

- **配色**：深色背景 `#0d1117` / `#161b22`，粉色强调 `#fb7299`

- **顶部栏**（`TopBar.tsx`）：
  - 搜索栏 + 搜索模式切换 `番名 | Tag`（Tag 模式下搜索标签名）
  - 五个分类按钮：在看、想看、搁置、看过、抛弃
  - 「新增番剧」按钮 → AniList 搜索弹窗

- **排序/Tag 筛选指示器**：网格上方显示当前排序条件和/或 Tag 筛选状态，可一键清除

- **番剧卡片网格**（`AnimeGrid.tsx`）：
  - Kazumi 风格，固定 6 列 `grid-template-columns: repeat(6, 1fr)`
  - 3:4 海报 + 底部番名 + 维度分数/BGM
  - 悬停上浮 + 粉色边框高亮
  - 「在看」卡片悬停显示删除按钮（仅隐藏本地，不同步 Excel）
  - 批量模式下卡片左上角显示复选框，点击切换选中，选中后粉色边框 + 发光

## 二、核心功能模块

### 1. 数据源与 Excel 交互

- **读取**：Vite 中间件 `/api/excel/read`，xlsx 库解析全部 Sheet
- **写入**：`/api/excel/write`，POST JSON 更新列表，仅修改可编辑列白名单
- **列映射**（`excelMapping.ts`）：30+ 列索引常量 + `DIMENSION_COL_MAP`
- **数据质量分类**：无维度评分或无评价 → 自动归入「在看」
- **分类覆盖**：用户手动修改后写入 localStorage 覆盖

### 2. 搜索与信息获取（`SearchAddModal.tsx`）

- **AniList GraphQL 代理**（`/api/anilist/search`）：返回封面、评分、年份、集数、标题
- **Bangumi 备用**（`/api/bangumi/search`）：依次尝试 `bangumi.online` / `api.bgm.tv`，8 秒超时
- **本地缓存**：`anilist_cache.json` / `bangumi_cache.json`，优先命中缓存
- **海报匹配**：启动时加载缓存，按标题模糊匹配关联海报；海报黑名单过滤误匹配
- **手动添加**：搜索无结果时可手动创建，id 前缀 `new-`（无 excelRowIndex）

### 3. 番剧详情面板（`AnimeDetailModal.tsx`）

**双模式设计**：默认查看模式（只读），点击「修改」进入编辑模式。

**上部** — Descriptions：上映日期、Bangumi 评分、制作组（编辑模式变为输入框）、分类选择器

**左侧 2/3**：
- **雷达图**（`RadarChart.tsx`）：ECharts 百分位排名雷达图，300px，粉色填充
- **维度评分**：4 列网格，标签 + InputNumber（电波精度 0.01，其余 0.1）+ 专项评价图标。编辑模式下可输入，查看模式下禁用
- **总评**：加权平均实时计算，右侧「导入评价」按钮（编辑模式下可见），按模板将全部维度评分+专项评价插入全局评价底部
- **维度专项评价**：点击 ✏️ 展开 TextArea，独立评语存储到 `DimensionReview[]`
- **全局评价**：TextArea，查看模式只读

**右侧 1/3**：
- **海报轮播**：3:4 容器，3 秒自动轮播，左右翻页，圆点指示器，当前封面标「封面」
- **海报拖拽**：mousedown/move/up 调整 `object-position`，持久化到 localStorage，拖拽时十字准星
- **按钮**：图片管理 / 💾 保存封面（仅外部 URL 时显示）/ 🔍 原图预览
- **角色标签**：Excel 角色列解析，紫色 Tag
- **Tag 编辑**：点击切换高亮，编辑模式下可删除和添加。输入为空时点击 `+` 弹出标签选择器（从全部番剧收集可选标签）

### 4. 雷达图（`RadarChart.tsx`）

- 排除总评维度，轴值为各维度百分位排名（0-100）
- `rankingService.ts`：`buildPercentileMap()` 统计全体分布，`getPercentileScores()` 查表
- 暗色透明背景，`#fb7299` 线条 + 15% 透明填充
- mini 模式支持 140px 缩略版

### 5. 图片管理（`ImageManager.tsx`）

- **存储**：本地文件系统 `images/{番剧名}/`，通过 Vite API 读写
- **命名**：`{番剧名}_{自增编号}.{扩展名}`
- **上传**：FileReader → Base64 → POST `/api/images/save`，直接保存原图不压缩
- **展示**：flex-wrap 布局，高度可配置，`objectFit: contain`，图片下方显示文件名
- **原图预览**：每张图使用 antd Image 组件，点击查看全分辨率原图
- **设为海报**：📌 → posterUrl 指向本地 API 路径，持久化到 IndexedDB
- **删除**：调用 `/api/images/delete` 删除本地文件
- **保存封面**：详情面板中外部封面（AniList CDN）可一键保存到本地

### 6. 维度管理（`DimensionManager.tsx`）

- 增删改维度 + 权重滑块（0-0.5，0.01 步长）
- 总和提示（建议 = 1.0），持久化到 localStorage

### 7. 单集评价（`EpisodeReview.tsx`）

- 剧集选择、主要剧情、名场面、单集评分（0-10）、观后感
- localStorage 存储 `anime_diary_episode_reviews`

### 8. 批量添加标签

- 侧栏 Tag 管理中点击「批量添加」进入批操模式
- 侧栏标签多选 + 主网格卡片多选
- 底部浮动状态栏显示「已选 X 个标签 → Y 部番剧」+ 确认/取消
- 确认后全部选中标签添加到全部选中番剧，同步写回 Excel

## 三、数据持久化

| 数据 | 存储位置 | 说明 |
|------|----------|------|
| 评分、评价、tag | Excel 文件 | Vite API 直写 |
| 分类覆盖 | localStorage `anime_diary_categories` | `{ animeId: category }` |
| 在看删除黑名单 | localStorage `anime_diary_watching_deleted` | `Set<animeId>` |
| 维度自定义 | localStorage `anime_diary_dimensions` | `Dimension[]` |
| 单集评价 | localStorage `anime_diary_episode_reviews` | `EpisodeReview[]` |
| 维度专项评价 | localStorage `anime_diary_dim_reviews` | `{ animeId: DimensionReview[] }` |
| 海报覆盖 URL | IndexedDB `anime_diary_poster_overrides` | 仅存 URL 字符串（非图片数据） |
| 用户图片 | 本地文件 `images/{番剧名}/` | Vite API 读写，自动命名 |
| 海报焦点位置 | localStorage `anime_diary_poster_positions` | `{ x, y }`（百分比） |
| 海报黑名单 | localStorage `anime_diary_poster_blacklist` | 误匹配海报不再加载 |
| 图片高度偏好 | localStorage `anime_diary_img_height` | 默认 360 |
| Tag 预设 | localStorage `anime_diary_tag_presets` | `string[]` |

### 一键导出/导入

设置区域提供 **💾 导出用户数据** 和 **📂 导入用户数据**。

- **导出**：将全部 localStorage + IndexedDB 数据 + `images/` 本地图片打包为一个 `.zip` 文件下载
- **导入**：上传 `.zip` 文件，自动解包恢复图片到 `images/{番剧名}/`，恢复所有配置并刷新

> ⚠️ Excel 文件需另外拷贝（📥 导出 Excel / 📤 导入 Excel）。

## 四、评分维度体系

| 维度 key | 标签 | 权重 | 说明 |
|----------|------|------|------|
| overall | 总评 | 0 | 各维度加权平均（不参与排序和雷达图） |
| audio | 音声 | 0.12 | OP/ED/BGM/声优表现 |
| production | 制作 | 0.15 | 分镜、演出、张数、运镜、特效、摄影 |
| animation | 作画 | 0.13 | 画面质感、构图、色彩与光影 |
| immersion | 沉浸 | 0.15 | 音乐、画面、分镜、剧情协同度、世界架构 |
| plot | 剧情 | 0.20 | 作品的灵魂 |
| character | 人设 | 0.10 | 角色设计 |
| depth | 深度 | 0.10 | 思想深度 |
| vibe | 电波 | 0.05 | 个人主观综合（**精度 0.01**，其余维度 0.1） |

**伪维度**：`bgm`（Bangumi 评分排序）、`namesort`（中文拼音排序，保持在当前分类）

## 五、排序系统

1. **维度排序**：过滤有该维度分数的条目，降序排列
2. **BGM 排序**：过滤有 Bangumi 评分的条目，降序排列
3. **番名排序**：`localeCompare('zh')`，支持正倒序，保持在当前分类下不跳转
4. **侧栏联动**：点击维度 → 主网格排序（番名外均切到「看过」）
5. **切换分类**：自动清除排序

## 六、数据导入/导出

- **导入**：支持 .xlsx/.xls，按 `EXCEL_COL` 常量映射字段
- **导出**：生成 40 列表头，全部数据写为 `番评分_导出.xlsx`
- **保存**：详情面板修改 → `ExcelUpdate[]` → `/api/excel/write`；Tag 增删改 → 遍历受影响条目写回

## 十、知识图谱

> 详见 [`知识图谱.md`](./知识图谱.md) 完整设计文档

基于 ECharts 力导向图的番剧关系网络可视化。从现有番剧数据中抽取制作公司、标签、角色作为实体节点，番剧通过实体节点间接关联，力导向布局自动聚拢关系紧密的番剧。

**入口**：侧栏「🔗 知识图谱」→ 点击「全屏查看」打开全屏 Modal

**核心交互**：

| 功能 | 说明 |
|------|------|
| 图浏览 | 拖拽节点 / 滚轮缩放 / 悬停高亮邻接 / 搜索定位 / 双击重置 |
| 关系筛选 | 按制作公司 / 标签 / 角色分别开关 |
| 连线模式 | 侧栏开关开启 → 点击番剧 → 点击实体节点 → 确认建立关联 |
| 数据写回 | 添加标签 / 设制作公司 / 添加角色 → 同步写入 Excel |

**视觉规则**：
- 实体节点颜色基于名称哈希，同名称始终同色
- 番剧默认灰色，吸收所连实体颜色——圈子归属一目了然
- 连接越多的节点越大，枢纽型番剧和热门标签自然突出

**文件**：`src/types/graph.ts`、`src/services/graphService.ts`、`src/components/KnowledgeGraphModal.tsx`

## 十二、已知问题与备注

- 生产模式下 Vite 中间件 API 不可用，降级为 Mock 数据
- AniList API 返回角色信息有限，角色列目前仅从 Excel 解析
- 导入 Excel 时不执行数据质量分类（与直接加载 Excel 行为不同）
- 维度管理保存后需刷新才能看到新增维度的评分输入框
- 总评列在 Excel 中为手动填入值，前端加权计算仅用于面板展示
- 图片存储在项目 `images/` 目录，换机器需同步该目录
