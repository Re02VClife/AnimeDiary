# 知识图谱 — 设计文档

> **更新日期**：2026-06-16  
> **技术栈**：ECharts 力导向图 + React 18 + TypeScript  
> **数据源**：从现有 `AnimeEntry[]` 就地抽取，无需额外存储

---

## 一、设计理念

将番剧列表从**扁平表格**升级为**关系网络**。番剧通过制作公司、标签、角色等实体节点间接关联，力导向布局自动将关系紧密的番剧聚拢，一眼看出品味偏好和圈子归属。

### 核心原则

| 原则 | 说明 |
|------|------|
| **零新依赖** | 复用 ECharts（已用于雷达图），不引入图数据库 |
| **就地抽取** | 从现有 `AnimeEntry` 数据实时构建，无持久化开销 |
| **间接关联** | 番剧之间不直接连线，通过实体节点间接聚拢，图干净可读 |
| **颜色叙事** | 实体节点颜色取自名称哈希，番剧颜色吸收所连实体的混合色 |

## 二、图数据模型

### 2.1 实体节点

| 节点类型 | 数据来源 | 聚合规则 | 节点颜色 |
|----------|----------|----------|----------|
| 🎬 番剧 | `AnimeEntry[]` | 直接映射 | 默认灰 `#6e7681`，吸收所连实体颜色 |
| 🏢 制作公司 | `entry.studio` | 出现 ≥2 次才建节点 | 名称哈希 → 色相，饱和度 55% |
| 🏷️ 标签 | `entry.tags[].name` | 出现 ≥2 次，最多 40 个 | 名称哈希 → 色相，饱和度 60% |
| 👤 角色 | `entry.characters[]` | 出现 ≥2 次，最多 30 个 | 名称哈希 → 色相，饱和度 50% |

### 2.2 关系边

| 边类型 | 方向 | 说明 |
|--------|------|------|
| `studio` | anime → studio | 番剧归属于制作公司 |
| `tag` | anime → tag | 番剧拥有该标签 |
| `character` | anime → character | 番剧包含该角色 |

> 番剧间关系（`same_studio`、`shared_tags`、`similar_scores`）已移除——通过实体节点间接聚拢即可表达相同信息，图更清爽。

### 2.3 节点 ID 命名空间

```
anime:{animeEntry.id}
studio:{公司名}
tag:{标签名}
character:{角色名}
```

## 三、视觉规则

### 3.1 节点大小（度数映射）

每种节点类型独立按度数（连接边数）线性映射到尺寸范围：

| 节点类型 | 最小尺寸 | 最大尺寸 | 默认（度数相同时） |
|----------|----------|----------|-------------------|
| 番剧 | 24px | 42px | 30px |
| 制作公司 | 14px | 24px | 18px |
| 标签 | 10px | 18px | 14px |
| 角色 | 10px | 18px | 14px |

### 3.2 颜色传播

```
番剧默认灰 (#6e7681)
   ↓
收集所连实体节点的 HSL 颜色
   ↓
取环形色相均值 + 65% 传播强度
   ↓
番剧染上实体混合色
```

- 无连接 → 纯灰
- 连接多个同色系实体 → 番剧呈该色系淡色
- 连接多色实体 → 番剧呈中性灰褐（多色抵消）

### 3.3 实体节点颜色

同一名称始终同色——基于名称字符串哈希，`hsl(nameHash % 360, 50~60%, 55%)`。

## 四、图形引擎（`graphService.ts`）

### 4.1 构建流程

```
buildGraph(animeList)
  │
  ├─ Step 1: 聚合实体节点
  │    ├─ 番剧节点：直接映射
  │    ├─ 制作公司：按 studio 分组，count ≥ 2 建节点
  │    ├─ 标签：按 tags 分组，count ≥ 2 建节点（最多 40）
  │    └─ 角色：按 characters 分组，count ≥ 2 建节点（最多 30）
  │
  ├─ Step 2: 生成关系边
  │    └─ 对每个番剧，连接其 studio / tag / character 实体节点
  │
  └─ 后处理
       ├─ applyNodeSizingByDegree() — 度数 → 尺寸
       ├─ applyColorPropagation() — 实体颜色 → 番剧颜色
       └─ 返回 { nodes, edges }
```

### 4.2 过滤函数

```typescript
filterGraph(data, visibleRelations)
```

按关系类型筛选边，自动移除孤立的非番剧节点，重新计算大小和颜色。

### 4.3 辅助函数

| 函数 | 用途 |
|------|------|
| `nameToHue(name)` | 字符串 → 0-360 色相 |
| `hslToHex(h, s, l)` | HSL → HEX |
| `hexToHsl(hex)` | HEX → HSL |
| `mixHslColors(colors)` | 多色环状均值混合 |
| `truncate(str, maxLen)` | 安全截断（用于节点标签） |

## 五、交互设计

### 5.1 视图模式

**入口**：侧栏「🔗 知识图谱」折叠区 → 点击「全屏查看」按钮 → 打开全屏 Modal（95vw × 82vh）

**布局**：

```
┌──────────────┬──────────────────────────┐
│ 左侧控制面板  │                          │
│ (260px)      │    ECharts 力导向图画布    │
│              │                          │
│ 🔍 搜索框    │   · 拖拽节点  · 滚轮缩放  │
│ ☑ 实体关系   │   · 悬停高亮  · 双击重置  │
│   标签/公司/  │                          │
│   角色       │                          │
│              │                          │
│ 节点图例     │                          │
│ 🔗 连线开关  │                          │
│ 操作提示     │                          │
├──────────────┴──────────────────────────┤
│ 底部详情条（选中番剧时显示）              │
└──────────────────────────────────────────┘
```

**标准操作**：

| 操作 | 效果 |
|------|------|
| 滚轮 | 缩放 |
| 拖拽空白 | 平移 |
| 拖拽番剧节点 | 移动位置 |
| 点击番剧 | 底部详情条 +「打开详情」按钮 |
| 悬停番剧 | 高亮邻接节点，其余淡化 |
| 搜索框 | 匹配节点高亮，其余半透明 |
| 筛选复选框 | 按关系类型开关边和实体节点 |
| 双击空白 | 重置视图 |

### 5.2 连线模式

通过侧栏滑动开关开启。开启后：

| 步骤 | 交互 | 视觉反馈 |
|------|------|----------|
| 1 | 点击番剧节点 | 源节点出现脉冲圈，粉色虚线连到鼠标 |
| 2 | 鼠标移向实体节点 | 距离 < 50px → 线变绿实线、实体被绿光圈磁吸高亮、横幅变绿提示 |
| 3 | 点击实体节点 | 弹出确认框 |
| 4 | 确认 | 数据更新（标签添加到番剧 / 设制作公司 / 添加角色），写回 Excel |

**RAF 循环**：连线模式下每帧计算鼠标到所有实体节点的几何距离，实现磁吸和高亮效果。

**设计理由**：不使用长按拖拽——ECharts 内置拖拽系统会拦截鼠标事件导致节点弹飞。改为开关 + 点击模式，连线时全局禁用 ECharts 拖拽，完全避开冲突。

## 八、连线模式的三种数据操作

| 目标类型 | 数据操作 | Excel 写回 |
|----------|----------|------------|
| 标签 | `entry.tags.push({ name, highlighted: true })` | TAG 列 |
| 制作公司 | `entry.studio = name` | STUDIO 列 |
| 角色 | `entry.characters.push(name)` | 仅内存（角色列固定位） |

注意：操作前检查重复——已有同名标签/公司/角色则静默跳过。

## 九、边界情况

| 场景 | 处理 |
|------|------|
| 番剧 < 3 部 | 显示空状态 "需要至少 3 部番剧" |
| 无 studio/characters 数据 | 对应节点不生成，无报错 |
| 筛选器清空所有边 | 番剧节点保留（孤立显示），实体节点移除 |
| Modal 不可见时渲染 | `destroyOnClose` + 打开时 `resize()` |
| 连线模式下切换筛选器 | 图正常 merge 更新，draggable 状态保持 |
| 目标已有同名标签 | `handleCreateRelation` 跳过重复 |
